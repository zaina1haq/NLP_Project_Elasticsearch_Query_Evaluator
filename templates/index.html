Elasticsearch Query Evaluator

An automatic rubric-based scoring system for Elasticsearch DSL queries. A fine-tuned Mistral-7B-Instruct model assigns scores 0–4 and generates structured rationales citing four independent criteria.

Mistral-7B-Instruct-v0.2 LoRA · rank 16 Reuters corpus QWK 0.957 Unsloth
Project Pipeline
1

Dataset Generation

Synthetic entries built from the Reuters corpus. Covers 8 query categories: content match, topic term, date range, bool, sort, nested, country filter, multi-match. All score levels 0–4 represented with balanced coverage.

2

Prompt Formatting

Entries formatted into Mistral [INST]…[/INST] instruction-tuning prompts. Each prompt includes the task, reference query, submission, and rubric. Expected output is a JSON object with score and rationale.

3

Baseline Evaluation

Base Mistral-7B evaluated on the 31-example test set: accuracy 54.8%, QWK 0.690. Over-predicts score 4 and under-predicts score 1 — a known bias of instruction-tuned models toward generous scoring.

4

LoRA Fine-Tuning

Unsloth + LoRA (rank 16, alpha 128) trained up to 10 epochs with early stopping (patience=3). All 7 attention and feed-forward projection layers targeted. ~40M trainable parameters (0.54% of total).

5

Post Fine-Tuning Evaluation

Fine-tuned model achieves 80.6% accuracy and QWK 0.957 on the held-out test set. Zero parse failures. Main remaining weakness: score 1 confusion (20% accuracy) — model sometimes assigns 0 or 2 instead of 1.

Score Scale
0
Completely invalid

Wrong top-level keys, non-existent fields, no valid operator.

1
Major errors

Recognizable query type but fundamentally broken (missing nested wrapper, critical wrong field).

2
Partially correct

Correct outer structure but a required clause is missing (no filter, no sort, wrong path).

3
Mostly correct

All clauses present, one minor issue (wrong field, imprecise operator, wrong sort direction).

4
Fully correct

Exactly right clause types, valid fields, correct operators, returns precisely the right documents.

Rubric Dimensions
QS
[Query Structure]

Is the DSL skeleton valid? Checks top-level keys, bool/must/filter nesting, and required wrappers (e.g. nested, bool).

FV
[Field Validity]

Do all field names exist in the Reuters mapping? Common errors: "headline" vs "title", "body" vs "content", "tags" vs "topics".

OC
[Operator Correctness]

Are match/term/range/nested used for the correct field types? "term" on analyzed text fields returns near-zero results.

TA
[Task Alignment]

Does the query retrieve what the task describes? A valid query can still fail by querying the wrong field or sorting the wrong direction.
